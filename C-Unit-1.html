Steps to Solve Problems

Problem-solving in programming involves a structured approach to develop efficient and effective solutions. It is the foundation for writing reliable code and building functional programs.

  1. Understand the Problem:

    Begin by reading and analyzing the problem statement carefully. Identify the given data (inputs), the desired result (outputs), and any constraints or special conditions. Misunderstanding the problem can lead to incorrect solutions.

  2. Plan the Solution:

    Before jumping into coding, it is essential to plan how to solve the problem. This includes selecting appropriate techniques like flowcharts, pseudocode, or algorithms. Consider edge cases and outline the step-by-step logic clearly.

  3. Write the Code:

    Translate the planned solution into a programming language such as C. Focus on proper syntax, variable declarations, control structures (if-else, loops), and functions. Aim for clean, readable, and modular code.

  4. Test the Code:

    Test your program with a variety of inputs, including typical, boundary, and invalid cases. This ensures the code behaves as expected and helps in validating the logic.

  5. Debug the Code:

    If the program doesnâ€™t work correctly, identify and fix errors. These could be syntax errors, logical errors, or runtime issues. Use debugging tools or insert print statements to trace the problem.

  6. Maintain the Code:

    Programs often need to be updated over time to fix bugs, improve performance, or add features. Writing well-documented and modular code helps with future maintenance.

Algorithms

An algorithmic approach is a systematic and step-by-step method for solving a problem or performing a task using a set of well-defined rules and instructions. Algorithms are fundamental to computer science and mathematics and play a crucial role in various fields, including programming, data analysis, and problem-solving.

Here are some key characteristics of algorithms:

  • Well-defined instructions: Each step is precise and executable.
  • Input and Output: It operates on input and produces output.
  • Finiteness: It must terminate after a finite number of steps.
  • Deterministic: Same input always results in same output.
  • Effective: Must be practical and clear to execute.
  • Step-by-step: Executed in a specific, logical order.
  • Optimality: Should ideally give the best result.
  • Problem-solving: Used to solve specific computational tasks.
  • Abstraction: Focuses on logic, not implementation detail.
  • Reusability: Applicable to similar problems with minimal changes.
  • Analysis: Assessed by time and space complexity.
  • Parallelism: Can be executed concurrently for efficiency.

Example 1: Algorithm to Find the Sum of Two Numbers

  1. Start
  2. Declare variables num1, num2 and sum
  3. Read values num1 and num2
  4. Add num1 and num2 and assign result to sum
  5. Display sum
  6. Stop

				#include <stdio.h>

				int main() {
					int num1, num2, sum;
					printf("Enter the first number: ");
					scanf("%d", &num1);

					printf("Enter the second number: ");
					scanf("%d", &num2);

					sum = num1 + num2;

					printf("The sum of %d and %d is %d\n", num1, num2, sum);

					return 0;
				}

Example 2: Algorithm to Find the Largest of Three Numbers

  1. Start
  2. Declare variables a, b and c
  3. Read values for a, b and c
  4. If a > b and a > c, display a is largest
  5. Else if b > c, display b is largest
  6. Else, display c is largest
  7. Stop

				#include <stdio.h>

				int main() {
					int a, b, c;

					printf("Enter three numbers: ");
					scanf("%d %d %d", &a, &b, &c);

					if (a > b && a > c) {
						printf("%d is the largest number.\n", a);
					} else if (b > c) {
						printf("%d is the largest number.\n", b);
					} else {
						printf("%d is the largest number.\n", c);
					}

					return 0;
				}

Pseudo Code

Pseudo code is a simplified way of describing an algorithm using plain English. It serves as a blueprint that outlines the logical flow of a program without being bound to the syntax of any programming language.

It's useful for planning and documenting logic that can be easily translated into real code. Even non-programmers with basic knowledge can understand pseudocode.

Advantages of Pseudo Code

  • Improves readability and planning.
  • Acts as a bridge between algorithm and code.
  • Works as documentation for future maintenance.
  • Reduces complexity during actual implementation.

How to Write Pseudo Code

  1. Write the main goal of the program.
  2. Declare variables clearly.
  3. Use indentation for control structures (if, loops).
  4. Use simple and clear naming conventions.
  5. Describe each step precisely without real syntax.

Program Development

Program development is the structured process of designing, writing, testing, and maintaining a software application. It involves several stages that ensure the software meets the user's needs and works efficiently and reliably.

  1. Requirement Analysis: This is the initial stage where developers gather all necessary information from stakeholders. The goal is to understand what the software must do and document the requirements clearly.
  2. Design: In this phase, the program's structure is planned, including its architecture, modules, and data flow. Diagrams like flowcharts and UML may be used.
  3. Coding: The actual program is written using a programming language (e.g., C, Python, Java). Clean, efficient, and well-commented code is essential.
  4. Testing: Once the code is written, it must be tested to catch and fix bugs. Testing ensures the software meets the requirements and performs as expected.
  5. Deployment: After testing is complete, the software is deployed to the production environment where users can access and use it.
  6. Maintenance: Post-deployment, the software needs updates and bug fixes to ensure long-term reliability. Maintenance also involves adding new features as needed.

Programming Languages and Generations

Programming languages have evolved significantly over time to simplify the process of writing software and to provide more power and flexibility to developers. These languages are typically categorized into five generations, each representing a step forward in abstraction and capability:

  • First Generation (1GL): These are machine-level languages, consisting entirely of binary code (0s and 1s). Programs are executed directly by the hardware without the need for a compiler or interpreter. While extremely fast, these are difficult and error-prone to write.
    Example: `10110000 01100001` (binary representation of a machine instruction)
  • Second Generation (2GL): Also known as assembly languages, these use symbolic names or mnemonics to represent machine instructions. They are slightly easier to understand than binary but still require detailed knowledge of the hardware.
    Example: `MOV AL, 61h` (moves the hexadecimal value 61 into register AL)
  • Third Generation (3GL): High-level programming languages that are more similar to human language. They are easier to learn and use and include structured syntax and abstraction from hardware. These languages require compilers or interpreters to convert code into machine language.
    Examples: C, Java, Python, Pascal, C++
  • Fourth Generation (4GL): Designed to be even closer to natural human language and to accomplish specific tasks with minimal code. Often used in database querying, report generation, and GUI development.
    Examples: SQL, MATLAB, Oracle Reports, Informix 4GL
  • Fifth Generation (5GL): These languages focus on problem-solving using constraints rather than explicit instructions. They are primarily used in AI and logic-based systems and rely on reasoning and rules.
    Examples: Prolog, Mercury, OPS5

As we move up the generations, the level of abstraction increases, making programming easier and more powerful, but often at the cost of lower control over hardware.

Introduction to C

C is a foundational programming language developed in the early 1970s by Dennis Ritchie at Bell Labs. It played a major role in shaping modern software development. Notably, C was used to implement the UNIX operating system, and it laid the groundwork for languages like C++, Java, and C#.

Why Learn C? C provides a deep understanding of how software interacts with hardware, making it an essential language for system-level programming and embedded systems development.

Key Features of C:

  • Procedural Language: Follows a linear, top-down approach where problems are broken into functions and solved step by step.
  • Mid-Level Language: Combines the low-level power of assembly with high-level programming constructs, offering both control and simplicity.
  • Rich Library: Includes standard libraries like `stdio.h`, `stdlib.h`, and `math.h` for input/output, memory handling, and math operations.
  • Manual Memory Management: Offers dynamic memory control through functions like `malloc()`, `calloc()`, and `free()`.
  • Pointers: Enables efficient data handling, especially useful in arrays, functions, and dynamic memory allocation.
  • Portability: Code written in C can be compiled and run on various hardware platforms with minimal changes.
  • Efficiency and Speed: C programs are close to hardware execution, making them ideal for real-time and performance-critical applications.

Applications of C: Operating systems, embedded systems, compilers, databases, drivers, and games.

Keywords and Identifiers

Keywords and identifiers are foundational building blocks of the C programming language. Understanding how to properly use them is essential for writing syntactically correct and meaningful programs. Keywords form the vocabulary of the language, while identifiers allow programmers to name and reference data and functions in their code.

Keywords

Keywords are reserved words predefined by the C language. These words have specific meanings and are used to perform various operations or define control flow, data types, and functions. Since keywords are integral to the language's syntax, they cannot be used as variable names or other identifiers.

Key Characteristics of Keywords:

  • Always written in lowercase (e.g., int, return).
  • Cannot be redefined or used as identifiers.
  • Essential for writing control structures, declaring data types, and more.

List of Common C Keywords:

					
						auto    break   case    char
						const   continue    default     do
						double  else    enum    extern
						float   for     goto    if
						int     long    register    return
						short   signed  sizeof  static
						struct  switch  typedef union
						unsigned    void    volatile    while

Examples of Keyword Usage:

  • int main(): int is a keyword declaring the return type.
  • if (x > 0): if is a control structure keyword.
  • for (int i = 0; i < 10; i++): for is a loop control keyword.
  • return 0;: return is used to return a value from a function.

Identifiers

Identifiers are the names assigned by the programmer to elements like variables, functions, arrays, structures, etc. Identifiers make code readable and maintainable, enabling programmers to refer to stored data and functions effectively.

Rules for Creating Identifiers:

  • Must begin with a letter (A-Z, a-z) or underscore (_).
  • Subsequent characters may include letters, digits (0-9), or underscores.
  • Cannot be the same as a keyword (e.g., int, float).
  • Cannot contain special characters such as @, #, $, etc.
  • C is case-sensitive, so count and Count are different.

Examples of Valid Identifiers:


						userName
						_age
						totalSum
						counter1
						MAX_LIMIT

Examples of Invalid Identifiers:


			1value      // Starts with a digit
			float       // Reserved keyword
			user-name   // Hyphen not allowed
			total sum   // Space not allowed

Best Practices for Naming Identifiers:

  • Be descriptive: Use names that convey purpose (e.g., studentScore).
  • Use consistent naming conventions:
    • Camel Case: totalMarks
    • Snake Case: total_marks
    • Pascal Case: TotalMarks
  • Use all caps for constants: #define MAX_SPEED 120
  • Avoid abbreviations unless standard: e.g., num for number is fine, but tst instead of test is unclear.

Summary:
Understanding the difference between keywords and identifiers helps prevent common syntax errors and improves code readability. Keywords are predefined, while identifiers are custom names created by the programmer. Always follow naming conventions and rules to write clean, professional, and bug-free code.

Variables and Data Types

Variables

Variable Declaration Syntax:


					data_type variable_name;

Examples of Variable Declarations:

Variable Initialization:

					data_type variable_name = initial_value;

Examples of Variable Initialization:

Constants

Constants in C are fixed values that do not change during the execution of a program. They are used to represent values that should remain constant throughout the program's lifetime. C provides two main ways to define constants: using the const keyword and using the #define preprocessor directive.

1. Using the const Keyword

The const keyword is used to declare a variable as a constant. Once a variable is declared with const, its value cannot be modified.

Syntax:

                    
                        const data_type variable_name = value;
  • const: Keyword used to declare a constant.
  • data_type: The data type of the constant (e.g., int, float, char).
  • variable_name: The name of the constant (following the rules for identifiers).
  • value: The constant value assigned to the variable.

Examples:

                    
                        const int MAX_VALUE = 100;
                        const float PI = 3.14159;
                        const char NEWLINE = '\n';

Key Points:

  • Constants declared with const follow the same scope rules as variables. They can be local or global.
  • It is a good practice to initialize const variables when they are declared.
  • Attempting to modify a const variable will result in a compilation error.

2. Using the #define Preprocessor Directive

The #define preprocessor directive is used to create symbolic constants. It is a preprocessor command, which means it is processed before the compilation phase. #define creates a macro that substitutes a specified value for every occurrence of the macro name in the code.

Syntax:

                    
                        #define CONSTANT_NAME value
  • #define: Preprocessor directive to define a constant.
  • CONSTANT_NAME: The name of the constant (conventionally written in uppercase).
  • value: The constant value.

Examples:

                    
                        #define MAX_SIZE 200
                        #define GREETING "Hello, World!"
                        #define TRUE 1
                        #define FALSE 0

Key Points:

  • #define constants are not variables; they are symbolic names that are replaced by their values before compilation.
  • #define constants do not have a specific data type. The data type is determined by the value assigned.
  • #define constants have global scope; they are visible throughout the entire program after their definition.
  • It is not necessary to end #define statements with a semicolon (;).

Differences between const and #define

Feature const #define
Type Declares a variable with a specific data type. Creates a symbolic name; no specific data type.
Scope Follows variable scope rules (local or global). Global scope from the point of definition.
Processing Processed during compilation. Processed by the preprocessor before compilation.
Memory Allocation Allocates memory like a variable. Does not allocate memory; performs text substitution.
Debugging Visible to the debugger. Not directly visible to the debugger (name is replaced by value).
Semicolon Requires a semicolon at the end of the declaration. Does not require a semicolon.

When to Use Which

  • Use const when you need a constant with a specific data type and scope control (e.g., local constants within a function).
  • Use #define for simple constant definitions that are used globally throughout the program and when you want preprocessor substitution (e.g., for simple macros).

I/O Statements

In C programming, input/output (I/O) statements are used to interact with the user or external devices. They allow the program to receive input from the user and display output to the user. The standard I/O functions in C are provided by the stdio.h header file.

1. Standard Output: printf()

The printf() function is used to display output to the standard output device (usually the console). It can display formatted output, including text, numbers, and characters.

Syntax:

                    
                        printf("format string", arguments);
  • "format string": A string that specifies the format of the output. It can contain:
    • Text to be displayed as is.
    • Format specifiers that indicate how to display the arguments.
    • Escape sequences for special characters.
  • arguments: Optional values to be inserted into the format string according to the format specifiers.

Format Specifiers:

Format specifiers are placeholders in the format string that are replaced by the values of the arguments. Common format specifiers include:

  • %d or %i: Integer
  • %f: Floating-point number (float)
  • %lf: Floating-point number (double)
  • %c: Character
  • %s: String

Escape Sequences:

Escape sequences are special character combinations that represent non-printable characters or perform specific formatting tasks. Common escape sequences include:

  • \n: Newline
  • \t: Horizontal tab
  • \\: Backslash
  • \": Double quote
  • \': Single quote

Examples:

                    
                        #include <stdio.h>

                        int main() {
                            int age = 30;
                            float salary = 50000.50;
                            char initial = 'J';
                            char name[] = "John Doe";

                            printf("Hello, %s!\n", name);
                            printf("Age: %d\n", age);
                            printf("Salary: %.2f\n", salary); // Display with 2 decimal places
                            printf("Initial: %c\n", initial);
                            printf("Age: %d, Salary: %.2f, Initial: %c\n", age, salary, initial);

                            printf("This is a tabbed line:\tTabbed text\n");
                            printf("This is a backslash: \\ \n");
                            printf("This is a double quote: \" \n");

                            return 0;
                        }

2. Standard Input: scanf()

The scanf() function is used to read input from the standard input device (usually the keyboard). It can read formatted input, such as numbers, characters, and strings.

Syntax:

                    
                        scanf("format string", &variable1, &variable2, ...);
  • "format string": A string that specifies the format of the expected input. It contains format specifiers that match the data types of the variables.
  • &variable1, &variable2, ...: Addresses of the variables where the input values will be stored. The ampersand (&) is the address-of operator and is necessary to modify the variable's value.

Format Specifiers:

The format specifiers used with scanf() are similar to those used with printf():

  • %d or %i: Integer
  • %f: Floating-point number(float)
  • %lf: Floating-point number (double)
  • %c: Character
  • %s: String (reads up to the first whitespace)

Examples:

                    
                        #include <stdio.h>

                        int main() {
                            int age;
                            float salary;
                            char initial;
                            char name[50]; // Array to store a string (max 49 characters + null terminator)

                            printf("Enter your age: ");
                            scanf("%d", &age);

                            printf("Enter your salary: ");
                            scanf("%f", &salary);

                            printf("Enter your initial: ");
                            scanf(" %c", &initial); // Note the space before %c to consume any leading whitespace

                            printf("Enter your name: ");
                            scanf("%s", name); // Reading a single word, use fgets for a line

                            printf("\nAge: %d, Salary: %.2f, Initial: %c, Name: %s\n", age, salary, initial, name);

                            return 0;
                        }

Important Notes:

  • Whitespace: scanf() generally ignores leading whitespace (spaces, tabs, newlines) when reading numbers and strings (%d, %f, %s). However, it does not ignore whitespace when reading a single character (%c). To avoid issues with %c, you can add a space before the format specifier in scanf() (e.g., scanf(" %c", &ch);).
  • Buffer Overflow: When using %s to read a string, you must ensure that the character array (e.g., name in the example) is large enough to hold the input. Otherwise, it can lead to a buffer overflow, which is a security vulnerability. Use fgets() for safer string input.
  • Return Value: scanf() returns the number of input items successfully read. You can use this return value to check for errors in input.
  • Newline Character: When you press Enter, a newline character (\n) is generated. If you don't handle this properly, it can cause unexpected behavior in subsequent input operations.

3. Character I/O: getchar() and putchar()

getchar() and putchar() are functions used for single character input and output, respectively.

getchar()

The getchar() function reads a single character from the standard input (usually the keyboard) and returns the ASCII value of the character as an integer.

Syntax:

                    
                        int getchar(void);

Example:

                    
                        #include <stdio.h>

                        int main() {
                            int ch;

                            printf("Enter a character: ");
                            ch = getchar();

                            printf("You entered: %c (ASCII value: %d)\n", ch, ch);

                            return 0;
                        }

putchar()

The putchar() function writes a single character to the standard output (usually the console).

Syntax:

                    
                        int putchar(int character);

Example:

                    
                        #include <stdio.h>

                        int main() {
                            char ch = 'A';

                            printf("Character to display: %c\n", ch);
                            putchar(ch);
                            putchar('\n'); // Newline character

                            return 0;
                        }

4. String I/O: fgets() and puts()

fgets() and puts() are functions used for string input and output, respectively.

fgets()

The fgets() function reads a line of text from the standard input (or a file) and stores it in a character array (string). It is safer to use than scanf() with %s because it limits the number of characters read, preventing buffer overflows.

Syntax:

                    
                        char *fgets(char *str, int n, FILE *stream);
  • str: Pointer to the character array where the string will be stored.
  • n: Maximum number of characters to read (including the null terminator).
  • stream: Pointer to the input stream (stdin for standard input).

Example:

                    
                        #include <stdio.h>

                        int main() {
                            char name[50];

                            printf("Enter your full name: ");
                            fgets(name, sizeof(name), stdin);  // Read up to 49 characters + null terminator

                            printf("Hello, %s", name); // puts() automatically adds a newline, fgets() might include one
                            // printf("Hello, %s", name);
                            return 0;
                        }

puts()

The puts() function writes a string to the standard output (usually the console) and appends a newline character at the end.

Syntax:

                    
                        int puts(const char *str);

Example:

                    
                        #include <stdio.h>

                        int main() {
                            char message[] = "Welcome to C programming!";

                            puts(message);  // Output the string
                            puts("This is another line."); // Output a string literal

                            return 0;
                        }

Operators, Precedence, and Associativity

Operators are symbols that perform operations on one or more operands (values or variables). C provides a rich set of operators that can be used to perform arithmetic, logical, relational, bitwise, and other operations. Understanding operator precedence and associativity is crucial for writing correct and efficient C programs.

1. Arithmetic Operators

Arithmetic operators are used to perform mathematical calculations.

Operator Description Example
+ Addition a + b
- Subtraction a - b
* Multiplication a * b
/ Division (integer division if both operands are integers) a / b
% Modulus (remainder of integer division) a % b

Example:

                    
                        #include <stdio.h>

                        int main() {
                            int a = 10, b = 5;
                            int sum, difference, product, quotient, remainder;

                            sum = a + b;
                            difference = a - b;
                            product = a * b;
                            quotient = a / b;
                            remainder = a % b;

                            printf("Sum: %d\n", sum);
                            printf("Difference: %d\n", difference);
                            printf("Product: %d\n", product);
                            printf("Quotient: %d\n", quotient);
                            printf("Remainder: %d\n", remainder);

                            return 0;
                        }

2. Relational Operators

Relational operators are used to compare two values. They return 1 (true) if the condition is true and 0 (false) if the condition is false.

Operator Description Example
== Equal to a == b
!= Not equal to a != b
> Greater than a > b
< Less than a < b
>= Greater than or equal to a >= b
<= Less than or equal to a <= b

Example:

                    
                        #include <stdio.h>

                        int main() {
                            int a = 10, b = 5;

                            printf("a == b: %d\n", a == b);
                            printf("a != b: %d\n", a != b);
                            printf("a > b: %d\n", a > b);
                            printf("a < b: %d\n", a < b);

3. Logical Operators

&& Logical AND (true if both operands are true) a && b
|| Logical OR (true if at least one operand is true) a || b
! Logical NOT (true if the operand is false, and vice versa) !a

4. Bitwise Operators

Bitwise operators perform operations on individual bits of data.

Operator Description Example
& Bitwise AND a & b
| Bitwise OR a | b
^ Bitwise XOR (exclusive OR) a ^ b
~ Bitwise NOT (one's complement) ~a
<< Left shift a << 2

Example:

                    
                        #include <stdio.h>

                        int main() {
                            unsigned int a = 60;  // 0011 1100 in binary
                            unsigned int b = 13;  // 0000 1101 in binary
                            unsigned int result = 0;

                            result = a & b;       // 0000 1100
                            printf("a & b = %d\n", result);

                            result = a | b;       // 0011 1101
                            printf("a | b = %d\n", result);

                            result = a ^ b;       // 0011 0001
                            printf("a ^ b = %d\n", result);

                            result = ~a;          // 1100 0011
                            printf("~a = %d\n", result);

                            result = a << 2;      // 1111 0000
                            printf("a << 2 = %d\n", result);

                            result = a >> 2;      // 0000 1111
                            printf("a >> 2 = %d\n", result);

                            return 0;
                        }

6. Increment and Decrement Operators

Increment and decrement operators are used to increase or decrease the value of a variable by 1.

Operator Description Example
++ Increment (increase by 1) a++ or ++a
-- Decrement (decrease by 1) a-- or --a

There are two forms of increment and decrement operators: prefix and postfix.

  • Prefix: The operator is placed before the variable (e.g., ++a). The value of the variable is modified before it is used in the expression.
  • Postfix: The operator is placed after the variable (e.g., a++). The value of the variable is modified after it is used in the expression.

Example:

                    
                        #include <stdio.h>

                        int main() {
                            int a = 5, b = 5;

                            printf("a++: %d\n", a++);  // Output: 5, then a becomes 6
                            printf("a: %d\n", a);      // Output: 6

                            printf("++b: %d\n", ++b);  // b becomes 6, then Output: 6
                            printf("b: %d\n", b);      // Output: 6

                            return 0;
                        }

7. Conditional Operator (Ternary Operator)

The conditional operator is a ternary operator (it takes three operands) that provides a concise way to express simple if-else statements.

Syntax:

                    
                        condition ? expression_if_true : expression_if_false;
  • condition: An expression that is evaluated to true (non-zero) or false (0).
  • expression_if_true: The expression that is evaluated and returned if the condition is true.
  • expression_if_false: The expression that is evaluated and returned if the condition is false.

Example:

                    
                        #include <stdio.h>

                        int main() {
                            int a = 10, b = 20;
                            int max;

                            max = (a > b) ? a : b;

                            printf("Max value: %d\n", max);

                            return 0;
                        }

Operator Description Example

, Comma operator (separates multiple expressions in a single statement) a = 1, b = 2

Example:

                    
                        #include <stdio.h>

                        struct Point {

Operator Precedence and Associativity

Operator precedence determines the order in which operators are evaluated in an expression. Associativity determines the direction (left-to-right or right-to-left) in which operators of the same precedence are evaluated.

Operator Precedence Table (Highest to Lowest):

Precedence Operator Associativity
1 () [] -> . Left-to-right
2 ! ~ ++ -- + (unary) - (unary) * (pointer) & sizeof Right-to-left
3 * / % Left-to-right
4 + - Left-to-right
5 << >> Left-to-right
6 < <= > >= Left-to-right
7 == != Left-to-right
8 & (bitwise AND) Left-to-right
9 ^ Left-to-right
10 | Left-to-right
11 && Left-to-right
12 || Left-to-right
13 ?: (conditional) Right-to-left
14 = += -= *= /= %= &= |= ^= <<= >>= (assignment) Right-to-left
15 , (comma) Left-to-right

Examples:

                    
                        #include <stdio.h>

                        int main() {
                            int a = 5, b = 10, c = 15, result;

                            result = a + b * c;      // Multiplication has higher precedence than addition
                            printf("a + b * c: %d\n", result);  // Output: 155 (5 + (10 * 15))

                            result = (a + b) * c;    // Parentheses override precedence
                            printf("(a + b) * c: %d\n", result);  // Output: 375 ((5 + 10) * 15)

                            result = c / a + b;      // Division and addition have same precedence, left-to-right associativity
                            printf("c / a + b: %d\n", result);  // Output: 13 ((15 / 5) + 10)

                            result = c / (a + b);    // Parentheses override associativity
                            printf("c / (a + b): %d\n", result);  // Output: 1 ((15 / (5 + 10))

                            result = a = b = c;      // Assignment has right-to-left associativity
                            printf("a = b = c: a=%d, b=%d, c=%d\n", a, b, c);  // Output: a=15, b=15, c=15

                            return 0;
                        }
